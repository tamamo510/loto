# GLEF - Gravity Loto Engine Framework

## このリポジトリの目的

GLEF（Gravity Loto Engine Framework）は、日本の数字選択式宝くじ「ロト6」「ロト7」の予測アプリである。
ブラウザで動作するHTML+JSの単一ファイルアプリ（index_v6.html）として、GitHub Pagesでホストされている。

最終目標は、予測精度を継続的に改善し、実際の購入で当選確率を高めること。
マイホーム資金のための長期プロジェクトである。

---

## 予測エンジンの概要

「4D Wave Physics」と呼ぶ独自の波形分析手法をベースに、複数の理論を組み合わせている。

### 波形エンジン（5成分）
- **depthWave** — ギャップ分析。各数字の出現間隔に基づくスコアリング
- **vertWave** — 時間軸同期。直近〜長期の出現頻度の重み付き評価
- **horzWave** — ゾーンMACD。ゾーン別出現頻度のトレンド分析
- **crossWave** — 共起行列。数字間の相関スコア（キャップ30）
- **coBias** — キャリーオーバー偏り。CO額による出現率の偏り

### 最適化エンジン
- **遺伝的アルゴリズム（GA）** — 波形スコアを適応度関数として、最適な組み合わせを探索
- **Auto-Tune** — 波形乗数5個をバックテスト結果に基づいて自動調整
- **シャノンエントロピー** — ゾーン分散度からcluster/neutral/spreadを判定し、GAの挙動を動的に変える

### 出力
- **ONE SHOT** — メイン予測（7個 or 6個）
- **ANTI-THEORY SHOT** — ONE SHOTとは異なる傾向の反セオリー予測

## 等級定義（公式準拠）

### ロト7
| 等級 | 条件 |
|------|------|
| 1等 | 本数字7個全て一致 |
| 2等 | 本数字6個一致 ＋ ボーナス数字2個のうち1個以上一致 |
| 3等 | 本数字6個一致 |
| 4等 | 本数字5個一致 |
| 5等 | 本数字4個一致 |
| 6等 | 本数字3個一致 ＋ ボーナス数字1個または2個一致 |

### ロト6
| 等級 | 条件 |
|------|------|
| 1等 | 本数字6個全て一致 |
| 2等 | 本数字5個一致 ＋ ボーナス数字1個一致 |
| 3等 | 本数字5個一致 |
| 4等 | 本数字4個一致 |
| 5等 | 本数字3個一致 |

## 作業ルール（全スレッド共通）

### 必ず守ること
1. **作業開始前にGLEF_PROGRESS.mdを読むこと。** 開発経緯・設計意図・現在の課題が書いてある
2. **バックテストでデータリークをしないこと。** N回の予測にはN-1回までのデータのみを使用する
3. **予測蓄積ファイル（GLEF_PREDICTIONS.jsonl）は上書きしない。** 修正のたびに新しいレコードを追記する
4. **修正したらGLEF_PROGRESS.mdに新しいversionとして記録すること。** 何を変えたか・なぜ変えたか・バックテスト結果を必ず含める
5. **等級定義は上記の公式定義に準拠すること。** ボーナス数字の扱いに注意
6. **作業完了後はcommit & push。** ブランチはmain直接で可

### やってはいけないこと
- バックテストでN回の本数字をN回の予測に使う（データリーク）
- GLEF_PREDICTIONS.jsonlの既存レコードを上書き・削除する
- GLEF_PROGRESS.mdの過去のversionの記録を改変する
- 設計意図を無視した修正を行う（迷ったらGLEF_PROGRESS.mdを参照）

### 作業の流れ
1. GLEF_README.md（本ファイル）を読む → 目的と構成を把握
2. GLEF_PROGRESS.mdを読む → 開発経緯と現在の課題を把握
3. 指示された作業を実行
4. バックテストで効果を検証
5. GLEF_PREDICTIONS.jsonlに予測結果を追記
6. GLEF_PROGRESS.mdに新versionを記録
7. commit & push

## データパイプライン

```
抽選結果の手入力（アプリUI）
    ↓
data.jsに自動追加
    ↓
新データで予測実行（4D Wave + GA）
    ↓
GLEF_PREDICTIONS.jsonlに自動追記
    ↓
次回抽選後に照合
    ↓
GLEF_RESULTS.jsonlに記録
    ↓
精度分析 → 次の修正方針
```

将来的にはhermesAgent（自律AIエージェント）がこのパイプラインを自動で回す予定。
現時点ではアプリUI上での手入力とClaudeCodeでの修正作業で運用する。
